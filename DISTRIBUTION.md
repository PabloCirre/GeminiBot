# 📦 GeminiBot Distribution Guide

This guide explains how to package GeminiBot into standalone, professional applications for macOS and Windows. 

## 📋 Prerequisites
- **Node.js** (v18 or higher)
- **npm** (v9 or higher)
- **icon.png** (at least 1024x1024) in the `assets/` folder.

---

## 🍎 Building for macOS
Since GeminiBot is built on Electron, the macOS build generates a `.dmg` installer and a `.zip` file for the current architecture (Intel or Apple Silicon).

### 1. Generate Native Icons
Run the provided script to create a high-resolution `.icns` file:
```bash
bash scripts/generate_icns.sh
```

### 2. Build the Bundle
```bash
npm run build:mac
```
**Output:** Look in the `dist/` folder for `GeminiBot-1.1.0-arm64.dmg`.

---

## 🪟 Building for Windows (PC)
The Windows build creates a professional **NSIS Installer** (`.exe`) with options for desktop shortcuts and custom installation paths.

### 1. Preparation
Ensure `electron-builder` is installed. The configuration in `package.json` already points to `assets/icon.png`.

### 2. Build the Installer
It is highly recommended to run this on a Windows machine:
```bash
npm run build:win
```
**Output:** Look in the `dist/` folder for `GeminiBot Setup 1.1.0.exe`.

---

## 🚀 Public Release (GitHub)
To distribute GeminiBot "massively" on GitHub:
1. Create a **New Release** on your GitHub repository.
2. Upload the files from the `dist/` folder (the `.dmg` and the `.exe`).
3. Point users to the download link in your `README.md`.

> [!IMPORTANT]
> **Code Signing:** These builds are currently **unsigned**. 
> - On macOS: Users may need to "Right-click -> Open" the first time.
> - On Windows: SmartScreen may flag the app as "Unknown Publisher".
> For a truly "pro" release, we recommend acquiring a Developer Certificate (Apple Developer / Certum for Windows).
